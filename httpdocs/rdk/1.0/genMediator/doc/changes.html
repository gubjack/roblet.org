Änderungen

Auf dieser Seite sind die Anpassungen, Erweiterungen und Änderungen aufgelistet, die für die vorliegende Version gemacht wurden.  Zuerst kommen die jüngsten (neuesten) Einträge.

Folgende Eintragsklassen gibt es:

  • Neue Dinge,
  • geänderte Sachen und
  • Fehlerkorrekturen.

Für Version 1.4

  • Ä Anpassungen für Quellverwaltung per SVN
  • Umstellung auf genRob®-genSite 3.3 für die Generierung der Dokumentation
  • Leicht mehr Ausgaben beim Start des Jini™-Reggie.
  • Wird der Eingabestrom geschlossen oder ein null-Gerät übergeben, so kommt keine Fehlermeldung mehr.

Für Version 1.3

  • N Neue Java™-Eigenschaft genRob.genMediator.httpd.port zur Angabe einer TCP-Port-Nummer für den eingebauten HTTP-Server (ohne Angabe bleibt es bei 8080)

Für Version 1.2

  • Umstellung der Quellen auf UTF-8
  • Umstellung des Baus auf aktuelle Ant-Scripte
  • Umstellung auf genSite 2.1.1

Für Version 1.1

  • Die Verzeichnisse zur Ablage temporärer Verzeichnisse und Dateien sind im Namen angepaßt worden.  Den Namen für die von genMediator angelegten Verzeichnissen wird nun der Nutzer angefügt.

    Dadurch muß ein für einen anderen Nutzer angelegtes Verzeichnis nicht mehr gelöscht werden, was z.B. unter Unix/Linus zu Komplikationen führte, da die passenden Schreibrechte i.d.R. nicht vorlagen.

Für Version 1.0

  • (1.0.1)
    • Einfügung unter Benutzung, wie man genMediator per Taste e beendet
  • Überarbeitete Dokumentation.
  • Start-Batch-Datei für den Browser hinzugefügt.
  • genRob®-genMediator benutzt jetzt die Eigenschaft java.io.tmpdir, um ein Verzeichnis zur Ablage temporärer Verzeichnisse und Dateien zu haben.  Damit können nun verschiedene Instanzen von verschiedenen Rechnern aus gestartet werden, auch wenn alle die gleichen (möglicherweise schreibgeschützten) Programmdateien benutzen.

Für Version 0.3

  • Hinzufügung des Dienstverzeichnis-Browsers (entlehnt aus den Jini™-Beispielen).
  • Umstellung der Ant-Datei auf die Nutzung von ../common.xml

Für Version 0.2

  • Der Jini™-Lookups gibt nun an Klienten die IP-Adresse als Host-Namen mit.
    Damit sollte ein Auffinden des Lookups auch möglich sein, wenn der Host-Name des Lookups dem Klienten nicht bekannt ist.

Für Version 0.1

  • Einführung von Ant
  • Starten der Komponenten nun per Java (genRob.genMediator.jar)
  • Dokumentation
  • Publikation bei genRob.com

Für Version 0.0

  • Initiale Version
  • Starten von RMID, HTTPD und Jini-Reggie per Batch/Script
