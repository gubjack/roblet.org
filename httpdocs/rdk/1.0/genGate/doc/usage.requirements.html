Einschränkungen

Der Roblet®-Server läßt in der vorliegenden Version einen unbeschränkten Zugriff auf den Wirtsrechner zu, was jedoch durch eine geeignete Sicherheitsrichtliniendatei vollständig korrigierbar ist.  Weiterhin gibt es meist direkt in den Betriebssystemen auch Kontrollmöglichkeiten - unter den Unix-artigen Betriebssystem käme z.B. chroot in Kombination mit der Firewall in Betracht.

Alle neueren Rechner jünger als 4 Jahre sollten genügen.  Bis auf CPU, Speicher, Dateisystem, Netzwerk und der Konsole werden keine Ressourcen verwendet.

Java™

Das Paket wurde unter "Java™ 2 Platform - Standard Edition" (wie von der Sun Microsystems, Inc., USA spezifiziert) in der Version 1.4.2 getestet.  Eine aktuelle Version kann für verschiedene Betriebssysteme von Sun's Java™-Website geholt werden.  Für Computer der Firma Apple sind nähere Informationen dazu hier zu finden - neuere Geräte (ab ca. Mitte 2004) sind bereits mit einer passenden Java™-Version ausgestattet.

Die Start-Dateien erwarten, daß sich das bin-Verzeichnis der Java™-Installation im Suchpfad für Programme befindet.

Einrichtung

Die Kombination genRob®-genControl/-genGate ist bis auf eine möglicherweise notwendige Parametrisierung sofort einsatzbereit.  Zweckmäßig ist die Nutzung und gegebenenfalls Anpassung der mitgegebenen Start-Dateien, anstatt bei jedem Start alle Parameter einzugeben.
Zu beachten ist, daß für einen produktiven Einsatz die originale Bibliothek des Herstellers von Jini™ verwendet werden muß.
