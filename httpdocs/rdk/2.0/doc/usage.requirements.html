Einschränkungen

Alle Rechner ab 2001 sollten genügen.  Vom Server werden bis auf CPU, Speicher, Dateisystem, Netzwerk und der Konsole keine Ressourcen verwendet.  Für die Arbeit mit einem Roblet®-Server ergeben sich in der vorliegenden Version noch einige Randbedingungen, die bei einem produktiven Einsatz zu beachten sind.

Java™

Das Paket wurde unter "Java™ 2 Platform - Standard Edition" (wie von der Sun Microsystems / Oracle, Inc., USA spezifiziert) in der Version 1.5.0 getestet.  Eine aktuelle Version kann für verschiedene Betriebssysteme von Sun's Java™-Website geholt werden.  Für Computer der Firma Apple sind nähere Informationen dazu hier zu finden - neuere Geräte sind bereits mit einer passenden Java™-Version ausgestattet.  Für andere Computer-Betriebssystem-Kombinationen bieten sich im Internet in der Regel passende Java™-Umgebungen.  Besonders erwähnt sei die auf der Sun-Implementierung basierende Variante OpenJDK.

Die in dieser Dokumentation erwähnten Batch- bzw. Script-Dateien erwarten, daß sich das bin-Verzeichnis der Java™-Installation im Suchpfad für Programme befindet ($PATH bzw. %PATH%).

Einrichtung

Nach Entpacken des Transport-Archivs (ZIP-Datei) ist das RDK für die Anwendungsentwicklung sofort einsatzbereit.  Eine Ausführung des Roblet®-Servers ist sofort möglich.  Eventuell sind Anpassungen hinsichtlich der Java™-Eigenschaften nötig.  Ähnliches gilt für das Server-Werkzeug und den Verzeichnisdienst.

Der Vertreterdienst ist bis auf eine möglicherweise notwendige Parametrisierung sofort einsatzbereit.  Zweckmäßig ist die Nutzung und gegebenenfalls Anpassung der mitgegebenen Start-Dateien, anstatt bei jedem Start alle Parameter einzugeben.

Zu beachten ist, daß für einen produktiven Einsatz die originale Bibliothek des Herstellers von Jini™ verwendet werden muß.
