Server

Der Roblet®-Server ist eine Java™-Anwendungen ohne grafische Oberfläche.  Er wird unter Unix, Mac OS X und Linux gewöhnlich in einem Terminalfenster mit einer Shell ausgeführt oder aus einem Script heraus gestartet.  Unter Windows kann man eine DOS-Box bzw. "Eingabeaufforderung" oder auch Cygwin verwenden.  Auch der Start über einen auf dem Desktop liegenden Verweis (Icon) auf eine Batch-Datei ist möglich.  Eine Ausführung als Daemon bzw. als Service ist bei geeigneter Umleitung der Standard-Ein- und -Ausgaben unter Zuhilfenahme eines passenden, Betriebssystems-abhängigen Zusatzprogramms ebenfalls möglich und durchaus erprobt.

Die Ausführung, d.h. der Start eines Servers, erfordert nur wenige Angaben.  Solche Angabe betreffen Server-Port, Log-Einstellungen, Module u.a.m.  Es empfiehlt sich der Übersichtlichkeit halber der Einsatz von Scripten bzw. Batches oder Start-Klassen.

Über die Befehlszeile kann im Terminalfenster bzw. in der Eingabeaufforderung mit dem Server kommuniziert werden.  Es kann darüber der Server regulär beendet werden.  Weiterhin sind diagnostische Kommandos verfügbar.

Ein Roblet®-Server verfügt über Mechanismen, die durchsetzen, daß die Sicherheit durch Roblets® nicht gefährdet werden kann.  Für die Arbeit mit einem Roblet®-Server ergeben sich in der vorliegenden Version noch einige Randbedingungen, die bei einem produktiven Einsatz zu beachten sind.
