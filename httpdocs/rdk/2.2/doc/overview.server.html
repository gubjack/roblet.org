Überblick

Im Rahmen der Roblet®-Technik wird eine Anwendung benötigt, die eine Laufzeitumgebung für Roblets® bietet, ihnen Ressourcen bereitstellt und zudem den Zugriff auf diese Ressourcen reguliert.  Diese Anwendung wird Roblet®-Server genannt.  Der wesentliche Teil des vorliegenden Paketes ist genau dieser Roblet®-Server.

Auf roblet®.org sind mehr Informationen zum Thema Roblet®-Technik zu finden.  Sind Sie neu in diesem Thema, so sollten Sie sich dort zunächst eine Übersicht verschaffen.  Die genannte Website dient auch der Zusammenarbeit von Interessierten und bietet weitere Kontakte.

Roblet®-Server inkl. Modul-Schnittstelle

Der Roblet®-Server in diesem Paket ist ein Java™-Programm ohne grafische Oberfläche.  Er kann mit Hilfe von Modulen parametrisiert werden, um Roblets® spezifische Ressourcen bereitstellen zu können:

Module müssen der Modul-Schnittstelle des Servers genügen.  Diese Schnittstelle ist einerseits für den Entwickler eine mit Javadoc erstellte Dokumentation und andererseits ein Java™-Archiv, welches der Compiler zum Zeitpunkt der Kompilation eines Moduls benötigt.  Dieses Archiv wird zur Laufzeit des Servers auch benutzt.

Werkzeug

Zur Betreuung von Roblet®-Server gibt es das Werkzeug. Es ist ebenso ein Java™-Programm ohne grafische Oberfläche:

Betreibt man das Werkzeug zur Fernwartung, so verschickt es zum fraglichen Server ein Roblet® und führt die gewünschten Aktionen aus bzw. holt Daten.  Das Werkzeug läßt sich aber auch zur Anzeige von Log-Dateien verwenden, die von Servern erzeugt wurden.  Für Verzeichnisse gibt es weitere Funktionen.

Die Nutzung des Werkzeuges ist hier beschrieben.
